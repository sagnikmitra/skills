# The self-check (mandatory before publishing)

This is the step that makes the skill reliable. A "casual tech-blogger" draft scored **38/100** against Sagnik's real openers because it drifted in predictable ways. Run this pass on every draft. If you have agent budget, do it adversarially: have a separate reviewer compare the draft sentence-by-sentence against `corpus.md` and score fidelity 0–100, listing drifts. Otherwise, self-audit honestly against the checklist.

## Hard fails (any one = rewrite that part)

- [ ] Opens with "Hey there" or any greeting tic.
- [ ] Contains a zinger / quotable one-liner / put-down (would survive as a standalone tweet).
- [ ] Snark or sarcasm aimed at a *person* or an *idea* (brands get a gentle wink; people and ideas do not).
- [ ] A crafted metaphor that shows off ("X wearing a Y costume") rather than a teaching analogy that explains.
- [ ] Anthropomorphises the system for sympathy ("the poor model...").
- [ ] Staccato hype: one-word sentences, hammer rhythm, "Boring. Deterministic."
- [ ] A setup→payoff comedy arc (demo-joy → betrayal → confession → reframe).
- [ ] Smooth-confessional self-deprecation ("more times than I'd like to admit") instead of concrete/awkward.
- [ ] Modern emoji, or more than ~1 ASCII emoticon, or more than ~1 aside per ~1000 words.

## Soft checks (should mostly be yes)

- [ ] Does it open by pre-empting the reader, a plain question, or thesis-first enthusiasm?
- [ ] Is it first-person, hedged with experience rather than authority?
- [ ] Does at least one teaching analogy build intuition for a hard concept?
- [ ] Is every strong claim paired with its honest caveat?
- [ ] Are sentences flowing and plain (not punchy, not ghostwriter-smooth)?
- [ ] Is there real depth — numbers, a schema, named tools, a formula — earning its place?
- [ ] Are sources credited and is there a hand-off link to the fuller version?
- [ ] Does it close gently (practical next-step / balanced verdict / soft sign-off), not on pure hype?

## The one-line test

Read each sentence and ask: *would this survive as a standalone tweet?* If yes — quotable, sharp, performed — soften it, bracket it, or cut it. His real best lines only work in conversation, cushioned by the sentences around them.

## Humanization tally (the "actually-him" pass)

Count these in the draft. If most are missing, it reads as a clean AI imitation, not as Sagnik — add them back (see `diction-and-tics.md`):

- [ ] A few **"pretty"**s (pretty well / good / much / easy).
- [ ] One or two **"to be honest" / "honestly"**.
- [ ] At least one **"in my usage" / "as of now" / "I think"** lived-experience hedge.
- [ ] A couple of sentences opening with **So, / Now, / Also, / Though, / Whatsoever,**.
- [ ] One genuine **exclamation** or simple-enthusiasm phrase ("a beast in itself", "goes like a charm").
- [ ] At least one bracketed **emoticon** ( :) :D :P ).
- [ ] One **reader-address or self-cross-link** ("you can always connect with me" / "I wrote the detailed thing in the paper →").
- [ ] One honest caveat in his words ("a mixed bag" / "not a deal-breaker or something" / "needs a little more work").
- [ ] A couple of **gently-loose** sentences — not ghostwriter-smooth. If every sentence is grammatically immaculate and evenly weighted, it is too clean for him.

## Calibration sanity

- If it reads like a LinkedIn thread or a conference keynote → too sharp, dial back to **subtle/warm**.
- If it reads like a dry spec with no warmth or analogy → too flat, the user explicitly does not want this; add teaching analogies and one or two gentle situational asides.
- The target is a knowledgeable friend explaining a hard thing over coffee: easy, playful, genuinely deep, and honest about the trade-offs.
